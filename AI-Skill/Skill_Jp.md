# 起業投資プロジェクト ウェブサイト コントリビュータースキル

## 概要

このリポジトリは、早稲田大学高等学院 公認団体「起業・投資プロジェクト」の**公式ウェブサイト**です。
**GitHub Pages** にて `https://kigyo-toshi.github.io/project/` でホストされています。

サイトは純粋な **HTML + CSS + バニラJavaScript** で構築されており、CDN経由で jQuery 3.4.1 を読み込んでいます。
ビルドステップやバンドラーは存在せず、ファイルがそのまま配信されます。

---

## このスキルの使用タイミング

以下の作業を行う場面でこのスキルを使用してください：

- **新しいページを追加**する場合（例：年度別活動ページの追加）
- **既存コンテンツの更新**（メンバープロフィール、FAQ項目、活動実績の記録など）
- **レイアウトやスタイルの不具合修正**
- **CSSのリファクタリング**（重複の削減やレスポンシブ対応の改善）
- **ナビゲーションの拡張**（新しいメニュー項目やSNSリンクの追加）
- **お問い合わせフォームの更新**やGoogle Formsの統合変更
- **アクセシビリティの改善**（ARIA属性、alt テキスト、セマンティックHTML）
- **SEOのメンテナンス**（Open Graphタグ、サイトマップ、メタディスクリプション）

## プロジェクト規約

### ファイル命名規則

| パターン | 用途 |
|---|---|
| `*.html` | 各HTMLファイルが独立したページ |
| `*.css` | 各ページに同じベースネームの専用CSSファイル |
| `accordtion.css` | 共有アコーディオンコンポーネントのスタイル（注：ファイル名にタイポあり） |
| `script.js` | 共有JavaScript（jQuery CDN + このファイルで読み込み） |
| `img/` | すべての画像アセット（SVG、PNG） |
| `sitemap.xml` | 検索エンジン用サイトマップ |

### HTML構造

すべてのページが以下のスケルトンに従います：

```
<!DOCTYPE html>
<html lang="ja">
  <head>
    <!-- メタブロック（charset、viewport、OGタグ、Google検証） -->
    <!-- ページ固有のCSSリンク -->
  </head>
  <body>
    <header>
      <section class="header-1200">  <!-- デスクトップナビ（≥1400px） -->
      <section class="header-768">   <!-- タブレット/モバイルナビ（<1400px） -->
    </header>
    <main>
      <!-- ページコンテンツ -->
    </main>
    <aside class=""></aside>          <!-- 現在未使用 -->
    <footer>
      <!-- ロゴ + ナビリンク + SNSリンク + 著作権表示 -->
    </footer>
    <!-- jQuery CDN -->
    <!-- script.js -->
  </body>
</html>
```

### レスポンシブブレークポイント

サイトは3段階のメディアクエリを使用しています（各CSSファイルで定義）：

| ブレークポイント | 対象 |
|---|---|
| `min-width: 1400px` | 大画面デスクトップ — `header-1200`表示、`header-768`非表示 |
| `1068px – 1399.98px` | タブレット / 小型デスクトップ — `header-768`に水平ナビ表示 |
| `max-width: 1067.98px` | モバイル — `header-768`にハンバーガーメニュー表示 |

### カラーパレット

| トークン | Hex値 | 用途 |
|---|---|---|
| 背景（暗） | `#1f1f1f` | ヘッダー、フッター |
| 背景（メイン） | `#2f2f2f` | メインコンテンツエリア |
| テキスト（白） | `#ffffff` | 本文テキスト、リンク |
| テキスト（淡色） | `#afafaf` | 見出し（h2、h3） |
| アコーディオン背景 | `#000000` | アコーディオンのボタン/パネル |

### ナビゲーションリンク

## 貢献時のベストプラクティス

### 新しいページの追加

1. 既存のページ（例：`information.html`）をテンプレートとしてコピーし、`newpage.html` を作成。
2. 対応するCSSをコピーして `newpage.css` を作成し、メインコンテンツのスタイルを調整。
3. **すべての既存HTMLファイル**のヘッダー/フッターナビゲーションに新しいリンクを追加。
4. `sitemap.xml` に新しいURLを追加。
5. アコーディオンを使用する場合、`<link rel="stylesheet" href="accordtion.css">` とインラインのアコーディオンスクリプトを含める。

### コンテンツの編集

- コンテンツは素のHTMLです — 該当ページの `<main>` セクションを直接編集。
- メンバープロフィールは `member.html` の `<div class="member2">` ブロック内に配置。
- FAQのアコーディオン項目は `faq.html` と `content.html` にあります — 既存の `accordion-item` パターンに従ってください。
- 活動実績は `achievement.html` に記載。

### CSSガイドライン

- **各ページが独自のCSSファイルを持つ。** レイアウト変更がグローバルに適用される場合は、すべてのCSSファイルを更新する必要があります。
- インラインスタイルの導入は避け、クラスベースのCSSを使用してください。
- 共有レイアウト（ヘッダー/フッター）を変更する場合、3つすべてのブレークポイントでテスト。
- 適切な場合は `clamp()` をフルイドサイジングに使用（`index.css` のヘッダーで既に使用済み）。

### アクセシビリティ

- インタラクティブ要素にARIA属性を使用（`content.html` / `faq.html` のアコーディオン参照）。
- `<img>` タグには必ず `alt` テキストを含める。
- セマンティック要素を使用（`<header>`, `<main>`, `<footer>`, `<nav>`, `<section>`）。

### テスト

- 各HTMLファイルをブラウザで直接開く（サーバー不要）。
- 幅 1400px以上、1068–1399px、1068px未満 でレスポンシブ動作を確認。
- 狭いビューポートでハンバーガーメニューの開閉が正しく動作するか確認。
- お問い合わせフォームが正しいGoogle Formsのエンドポイントに送信されるか確認。

### デプロイ

サイトはリポジトリのデフォルトブランチからGitHub Pagesにより自動デプロイされます。
変更をプッシュすれば、数分以内にサイトが更新されます。
