# .content — サイトコンテンツ管理ディレクトリ

このディレクトリは、サイト全ページのテキストコンテンツを**一元管理**するための場所です。

## 目的

- サイトの全コンテンツを**テキストベース**でまとめ、AIが参照・更新できるようにする
- HTMLファイルを直接読まなくても、サイトの内容を把握できる「**信頼できる唯一の情報源（Single Source of Truth）**」として機能する

## 使い方

### コンテンツを更新したい場合

1. このディレクトリ内の該当ファイルを編集する
2. 編集内容に基づいて、対応するHTMLファイルを更新する

### AIへの指示例

- 「メンバー情報を更新して」→ `members.md` を編集後、`member.html` に反映
- 「FAQ を追加して」→ `faq.md` を編集後、`faq.html` に反映
- 「SNSリンクを変更して」→ `site.md` を編集後、全HTMLファイルのヘッダー/フッターに反映

## ファイル構成

| ファイル | 内容 | 対応HTML |
|---|---|---|
| `site.md` | サイト全体の共通情報（SNS、ナビ、メタ等） | 全ページ共通部分 |
| `index.md` | トップページのコンテンツ | `index.html` |
| `content.md` | 活動内容ページ | `content.html` |
| `achievement.md` | 活動実績ページ | `achievement.html` |
| `members.md` | メンバー・OBページ | `member.html` |
| `information.md` | プロジェクト情報ページ | `information.html` |
| `faq.md` | よくある質問ページ | `faq.html` |
| `contact.md` | お問い合わせページ | `contact.html`, `contact-thankyou.html` |

## ルール

### 基本ルール

- **このディレクトリのファイルがコンテンツの正** — HTMLと内容が食い違う場合、こちらが正しい
- コンテンツ変更は必ずこのディレクトリを先に更新し、その後HTMLに反映する
- HTMLの構造（タグ、クラス名）はここでは管理しない。テキスト内容のみを管理する

### 体裁の整形ルール（重要）

**このディレクトリのファイルは、ユーザーが雑に・ラフに書くことを前提とする。**
AIがファイルを読み込む際は、以下の整形を必ず行うこと：

1. **読み込み時の整形**
   - マークダウンの見出しレベル（`#`, `##`, `###`）が乱れていたら統一する
   - 箇条書きや表のフォーマットが崩れていたら正しいMarkdown記法に直す
   - 半角・全角スペースの混在、不要な空行、インデントのずれを修正する
   - 情報が足りない場合は `（未記入）` と明記する

2. **HTML反映時の品質保証**
   - `.content` ファイルの内容がラフでも、HTMLに反映する際は正しい構造・体裁で出力する
   - 例：ユーザーが `faq.md` に「Q: 兼部できる？ A: できます」とだけ書いても、HTMLにはアコーディオンの正しい構造（`button`, `aria-expanded`, `role` 属性等）で反映する

3. **整形後の書き戻し**
   - 読み込んだファイルの体裁が崩れていた場合、整形した内容で `.content` ファイル自体も上書き更新する
   - これにより、次回以降の参照時にもきれいな状態が維持される
